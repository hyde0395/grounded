# grounded v0.2 (Shell-Write Gating + G-2 Package Verification) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Gate Bash-mediated file writes (`sed -i`, `tee`, `>`/`>>`) against the read ledger, and block installs of packages that don't exist on their registry (G-2) — the two areas Claude Code's built-in guard does not cover at all.

**Architecture:** Two new modules slot into the existing component layout: `shell_scan.py` (pure command parsing — write targets + package specs) and `registry.py` (registry existence lookup with injectable opener). `verdict.py` gains two pure decision functions; `pre_gate.py` gains a Bash branch that aggregates STOP/WARN verdicts, caches registry results in the ledger (`known_pkgs`, both true and false per spec §06), and emits WARN via PreToolUse JSON `additionalContext` (verified mechanism: exit 0 + `hookSpecificOutput.permissionDecision: "allow"` + `additionalContext` → model-visible, non-blocking).

**Tech Stack:** Python 3 stdlib only. unittest. Registries: npm `registry.npmjs.org/{name}`, PyPI `pypi.org/simple/{name}/`, crates.io `crates.io/api/v1/crates/{name}` (UA header required).

**Verdict policy (spec §05, 오탐 < 누락):**
- truncate (`>`)/in-place (`sed -i`, `perl -i`, `tee`) on existing unread file → **STOP**; append (`>>`, `tee -a`) → **WARN**; new file / read file / unresolvable target (vars, substitutions, quoted) / `/dev/*` → **PASS**.
- package found → PASS (cache true); 404/410 → **STOP** (cache false); network error/403/timeout → **PASS quietly**, no cache (retry next time). Max 5 lookups per command, 2.5s timeout each.
- Compound `cat f && sed -i … f` still STOPs: the model issues sed before it ever sees cat's output — genuinely ungrounded.
