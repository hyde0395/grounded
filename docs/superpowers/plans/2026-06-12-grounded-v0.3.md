# grounded v0.3 (G-3 URL Liveness) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Block fetches of demonstrably dead URLs (404/410/DNS failure) and warn on ambiguous ones (403/5xx/timeout), per spec §04 G-3 — covering both the WebFetch tool and `curl`/`wget` in Bash.

**Architecture:** Mirrors the G-2 pattern exactly. New `hooks/urlcheck.py` (network I/O, injectable opener, tri-state-ish result: HTTP status int / 0 for DNS-dead / None for unknown) + `shell_scan.fetch_urls` (pure parsing) + `verdict.gate_url` (pure mapping) + pre_gate branches for WebFetch and Bash URLs. Definitive results (alive 2xx-3xx, dead 404/410/0) cached in ledger `verified_urls` (spec §06 example: status ints); ambiguous results never cached so transient failures self-heal. post_record accrues successful WebFetch URLs as 200.

**Verdict policy:** 2xx/3xx → PASS · 404/410/DNS(0) → STOP · everything else incl. None → WARN (spec §05/§07: bot walls must not block). Skip checks entirely for: non-GET curl (`-X`, `-d`, `--data*`, `-F`, `-T`, `--upload-file`, wget `--post-*`/`--method`), private/loopback hosts (localhost, 127.*, 10.*, 192.168.*, 172.16-31.*, *.local, ::1). Max 3 URL pings per command, 2.5s timeout, HEAD with GET retry on 405/501.
